# 스포일러 노출 정책

## 1. 기본 원칙

| 원칙 | 정책 |
|---|---|
| 점수 비노출 | `watch_score`, `pregame_score`, `peak_base_score`는 숫자와 등급 모두 화면에 노출하지 않는다. |
| 서버 강제 | 보호 모드 필터링은 프론트가 아니라 API 응답 생성 단계에서 적용한다. |
| 결과 차단 | 점수, 승패, 리드 팀, 득점 결과, 결말을 직접 드러내지 않는다. |
| 짧은 설명 | 사용자는 계산 근거 전체가 아니라 진행 카드의 최신 태그, 종료 경기 한 줄 문구, 이벤트 타임라인 문구를 본다. |
| AI 제한 | AI는 추천 여부를 판단하지 않고, 서버가 전달한 안전한 context로 종료 경기 헤드라인과 이벤트 문구만 만든다. |

## 2. 모드별 노출 기준

### 보호 모드

보호 모드는 경기를 볼지 판단하는 데 필요한 맥락만 제공한다.

| 노출 가능 | 예시 |
|---|---|
| 매치업 | 팀 이름, 팀 약어 |
| 경기 상태 | 예정, 진행 중, 종료 |
| 진행 이닝 | 이닝 숫자 (초/말은 공격팀을 드러내므로 비노출) |
| 상황 맥락 | 아웃 카운트, 볼카운트, 득점권, 만루 |
| 추천 정보 | 진행 카드 최신 태그 1개, 종료 경기 AI 헤드라인, 종료 카드 `keyMoment`, 이벤트 타임라인 AI 문구 |
| 표시 정보 | 경기장, 시작 시각, 선발 투수, 관심 팀/선수 여부 |

팀 이름은 매치업 판단에 필요한 기본 정보이므로 보호 모드에서도 노출한다. 단, 어느 팀이 앞서거나 유리한지는 드러내지 않는다.

### 공개 모드

사용자가 경기 단위 공개 전환을 선택하면 아래 정보를 제공한다. 공개 전환은 경기 상세 화면 상단의 보호/공개 세그먼트 토글에서 별도 확인 절차 없이 즉시 적용되며, 해당 경기 상세에만 적용한다.

| 공개 가능 | 예시 |
|---|---|
| 점수 | 현재 점수, 최종 점수, play 전후 점수 |
| 결과 | 승패, 득점 play, 타석 결과, 원문 중계 텍스트 |
| 방향성 | 홈/원정 우세, 리드 변경 |
| 상세 매치업 | 타자/투수 이름, 이닝 초/말, play 타임라인 |

## 3. 경계 규칙

- **이닝 숫자는 노출, 초/말(`inningType`)은 비노출.** 이닝 숫자는 남은 관전 분량 판단에 필요하고 결과 방향을 드러내지 않는다. 초/말은 공격팀을 드러내 주자 압박과 결합 시 방향이 노출된다.
- **진행 중 경기의 최근 흐름은 진행 카드의 `latestTag`와 이벤트 타임라인으로만 제공한다.** 상세 화면의 태그 칩, 태그 변화 히스토리, `liveUpdateBlocks`, "흐름 변화" 섹션은 제공하지 않는다.
- **이벤트 태그(보호 표기 라벨)는 이벤트 타임라인 리스트의 `copy` 폴백으로만 쓰인다.** 펄스 그래프(`tensionCurve`)에는 이벤트 라벨·이벤트 마커를 표시하지 않는다. 그래프는 구간별 긴장 수준만 표현하고, 사건 표시는 이벤트 타임라인이 전담한다.
- **내부 추천 점수는 모드와 무관하게 항상 금지.** 공개 모드에서 공개되는 것은 "경기의 점수"이지 "추천 점수"가 아니다.
- **긴장 곡선(`tensionCurve`)은 다운샘플·양자화 조건을 만족할 때 허용되는 파생 예외다.** 서버가 `base_score`를 이닝 단위 이상으로 다운샘플하고 1~5 단계로 양자화한 값만 내려준다. **종료 경기에서만 제공한다**(보호 모드는 이닝 단위, 공개 모드는 하프이닝 단위까지). 진행 중 상세는 보호·공개 모두 `tensionCurve`를 제공하지 않는다. 원 점수 수치·시계열 원본·축 눈금 숫자는 어떤 모드에서도 노출하지 않는다. 단계는 경기별 절대 스케일 0~100 기준 양자화라 경기 간 비교 지표로 쓰지 않는다.

## 4. 화면별 노출 필드

| 화면 | 노출 | 숨김 |
|---|---|---|
| 홈 카드 (예정) | 매치업, 시작 시각, 구장, 선발 투수 | 추천 태그, `pregame_score`, 배당 |
| 상세 (예정) | 매치업, 시작 시각, 구장, 선발 투수, 관심 선수 선발 여부 | 추천 태그, 내부 점수·배당 일체 |
| 홈 카드 (진행) | 매치업, 이닝 숫자, LIVE 배지, `latestTag` 1개(nullable) | 점수, 초/말, 아웃카운트, `tags` 배열 |
| 홈 카드 (종료) | 매치업, AI 헤드라인, `keyMoment`(headline 없을 때 표시, nullable) | 최종 점수, 승패, 추천 태그 배열 |
| 상세 (진행, 보호) | 매치업·선발, 이닝 숫자, `situation`(아웃·볼·스트라이크·주자), 이벤트 타임라인(보호 안전 이벤트 + `copy`), `switchSuggestion` | 점수, 초/말, 현재 타자/투수, play 원문, 타석 결과, 태그 칩, 태그 히스토리, `liveUpdateBlocks`, 공개 전용 이벤트, `tensionCurve`(진행 중에는 미제공) |
| 상세 (진행, 공개) | 보호 항목 + 점수, 이닝 초/말, 현재 타자/투수(관심 선수면 강조 표시), play 타임라인(원문), 타석 결과, 이닝별 점수, 공개 전용 이벤트와 근거 수치 | 내부 추천 점수, `tensionCurve`(진행 중에는 미제공) |
| 상세 (종료, 보호) | 종료 경기 AI 헤드라인(보호 모드용), `tensionCurve`(이닝 단위, 노드 없는 순수 곡선), 이닝별로 그룹화한 이벤트 타임라인(보호 안전 이벤트 + `copy`) | 최종 점수, 승패, scoring_summary, 태그 칩, 공개 전용 이벤트, 펄스 그래프 이벤트 노드·라벨 |
| 상세 (종료, 공개) | 종료 경기 AI 헤드라인(공개 모드용, 최종 결과 반영 가능), 최종 점수, 이닝별 점수, scoring_summary(득점 play 목록), `tensionCurve`(하프이닝 단위 허용), 이벤트 타임라인(공개 문구·공개 전용 이벤트·근거 수치) | 내부 추천 점수 |

홈 하단 전체 경기 목록 카드는 상태별로 위 홈 카드(예정·진행·종료)와 동일한 노출·숨김 기준을 적용한다. 라이브 수집 이전 종료 경기처럼 헤드라인·대표 이벤트가 없는 카드는 매치업만 표시한다.

## 5. 금지 필드 목록 (직렬화 가드 대상)

| 범위 | 금지 필드 |
|---|---|
| 전 모드 | `watch_score` · `base_score` · `pregame_score` · `peak_base_score` · `signal_contributions` 등 내부 점수 일체, odds 전체. 단, `tensionCurve`는 아래 조건을 만족할 때만 파생 예외로 허용 |
| 보호 모드 | `runs`/`hits`/`errors`, 이닝별 점수, `inningType`(초/말), play `text`, `scoring_play`/`score_value`, 타석 `result`, `scoring_summary`, `home_score`/`away_score`, `game_events`의 `REVEALED_ONLY` 행 전체·`payload`·`inning_type` |

금지 판정은 값이 아니라 **키 기준**이다. 금지 필드는 직렬화 결과에서 키 자체가 사라져야 하며, `"finalScore": null`처럼 값만 비운 형태는 허용하지 않는다. 키가 남으면 응답 스키마만으로 결과 존재 여부를 추론할 수 있고, 계약 검증 측이 필드 존재를 위반으로 판정할 수 있다. 적용 범위는 클라이언트 응답 DTO와 scorer → ai-service 요청 `safeContext` 두 곳이다. 강제 수단은 모드별 DTO 분리 또는 해당 DTO에 대한 `@JsonInclude(NON_NULL)`이며, 전역 Jackson 설정에 의존하지 않는다.

예외는 공개 모드의 `winner`다. 무승부일 때 `null`이 "무승부"라는 사실을 뜻하므로 키를 유지한다. 보호 모드에는 이 키 자체가 없다.

`tensionCurve`는 전 모드 금지 필드 목록에서 유일하게 허용하는 `base_score` 파생 예외다. 서버가 원본 시계열을 이닝 단위 이상으로 다운샘플하고 1~5 단계로 양자화한 뒤 별도 필드로 내려줄 때만 노출할 수 있다. **종료 경기에서만 전체 경기 범위로 노출하며, 진행 중(보호·공개 모두)에는 노출하지 않는다.**

## 6. 금지 표현

보호 모드, 알림 문구, AI 헤드라인, 이벤트 문구에는 아래 표현을 사용하지 않는다.

| 금지 표현 | 이유 |
|---|---|
| 홈런 발생 | 구체적인 결과를 드러낸다. |
| 역전 성공 | 어느 팀의 상황인지 추측할 수 있다. |
| 끝내기 | 경기 결말을 드러낸다. |
| 실점 위기 | 특정 팀이 불리하다는 방향을 드러낸다. |
| 홈팀/원정팀 리드 | 앞서는 팀을 드러낸다. |
| 점수 차가 좁혀짐 | 점수 흐름과 방향을 드러낸다. |

## 7. 추천 이유 태그

태그는 내부 신호를 보호 모드에서 읽을 수 있는 짧은 표현으로 바꾼 값이다. AI 문구 생성 컨텍스트에는 내부 신호명이 아니라 이 표의 보호 표기 문자열만 전달한다.

| 내부 신호 | 보호 모드 태그 | 사용 위치 |
|---|---|---|
| `late_or_extra` | 후반 긴장 구간 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `score_gap` | 접전 흐름 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `recent_score` | 흐름 변화 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `lead_change` | 흐름 급변 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `big_inning` | 한 이닝 흐름 집중 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `pressure` | 득점권 압박 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `early_slugfest` | 초반 난타 흐름 | 진행 카드(`latestTag` 후보), 알림 템플릿 |
| `favorite_team` | 관심 팀 | 알림 템플릿 전용 |
| `standings_race` | 순위 경쟁 | 내부 전용(`pregame_score` 입력), 화면 태그 미사용 |
| `probable_pitcher_matchup` | 선발 맞대결 | 내부 전용(`pregame_score` 입력), 화면 태그 미사용 |

진행 카드에는 `latestTag` 1개만 노출한다. 기준은 기여도순이 아니라 가장 최근에 활성화된 태그이며, 후보는 `사용 위치`가 진행 카드인 태그로 제한한다. 응답에는 `tags` 배열을 포함하지 않는다.

`recent_score`의 보호 표기는 항상 `흐름 변화`로 유지하고, `점수`라는 단어가 들어간 표시명으로 바꾸지 않는다.

`count_pressure`, `pitcher_instability`, `hard_contact` 태그는 폐지한다. 각각 §8의 동명 또는 대응 이벤트(`full_count_two_out`, `pitcher_instability`, `hard_contact`)로 대체하며, 상세 화면에서는 이벤트 타임라인(모든 상태)으로 표현한다. `favorite_player` 태그는 폐지하고, 공개 모드의 현재 타자/투수·최근 플레이 강조 표시로 대체한다(보호 모드는 현재 타자/투수를 노출하지 않으므로 강조도 나타나지 않는다).

`standings_race`는 `standings` 유래, `probable_pitcher_matchup`은 `player_season_stats` 유래 내부 입력이다. 배당(`odds`) 유래 태그는 보호 모드와 공개 모드 모두에서 만들지 않는다.

## 8. 흥미로운 순간 이벤트

경기 상세 타임라인에 표시하는 순간 단위 이벤트다. 펄스 그래프와는 독립적으로 표시하며 그래프 노드로는 쓰지 않는다. scorer가 라이브 계산 중 임계를 통과한 순간을 추출해 `game_events`에 영속하며, 종료 경기에도 그대로 재사용한다. 태그(§7)가 진행 카드와 알림용 경기 단위 상태 표기라면 이벤트는 발생 시점이 고정된 점 단위 표기다.

| 이벤트 | 발생 조건(초기값) | 원천 | 등급 | 보호 표기 | 공개 표기 |
|---|---|---|---|---|---|
| `pressure_bases_loaded` | 만루 타석 | plays `runner_on_*` | 보호 안전 | 만루 승부 | 만루 승부 |
| `pressure_scoring_position` | 2·3루 주자 + 2아웃 | plays `runner_on_*` | 보호 안전 | 득점권 승부 | 득점권 승부 |
| `long_at_bat` | 한 타석 8구 이상 | PA `pitch_number` | 보호 안전 | 긴 접전 승부 | 긴 접전 승부 |
| `full_count_two_out` | 풀카운트 + 2아웃 | plays 카운트 | 보호 안전 | 승부처 카운트 | 승부처 카운트 |
| `pitcher_instability` | 선발 100구 이상 또는 구속 2mph 이상 하락 | PA `pitcher_pitch_count`·`release_speed` | 보호 안전 | 투수 흔들림 | 투수 흔들림 |
| `hard_contact` | `is_barrel` 또는 `exit_velocity >= 100` | PA | 보호 안전 | 강한 타구 | 강한 타구 |
| `scoring_play` | 득점 play | plays | 공개 전용 | — | 득점 |
| `lead_change` | 앞선 팀 변경 | plays | 공개 전용 | — | 리드 교체 |
| `home_run` | 홈런 | plays | 공개 전용 | — | 홈런 |
| `big_inning` | 한 이닝 득점 play 2개 이상 | plays | 공개 전용 | — | 빅이닝 |

- 보호 표기에는 결과·방향성 표현(§6)을 결합하지 않는다. 선수명은 보호 모드에서 표기하지 않는다.
- 라벨 매핑은 모드별로 분리한다(`GameEventLabelPolicy`). 보호 라벨은 `PROTECTED_SAFE` 이벤트에만 존재하며, 판정은 라벨 존재 여부가 아니라 `spoiler_level`을 먼저 검사한다. 미지 `event_type`은 두 모드 모두 라벨 없음(기본 차단)이다.
- 발생 임계와 경기당 상한(`hard_contact` 타석당 1회·경기당 8회 등)은 `scoring.yml`에서 관리한다. 상한은 타임라인 노이즈와 이벤트 밀집에 의한 간접 스포일러를 막기 위한 값이다.
- 관심 선수 타석은 사용자별 데이터이므로 전역 이벤트로 두지 않고 조회 시점에 계산한다.

### 이벤트 AI 문구 생성·저장 규칙

- scorer는 `game_events` 영속 직후 `POST /ai/event-copy`를 비동기로 요청한다. 보호 안전 이벤트는 `mode=PROTECTED`와 `mode=REVEALED`를 각각 요청하고, `REVEALED_ONLY` 이벤트는 `mode=REVEALED`만 요청한다.
- `spoilerSafe=true`, `fallbackUsed=false`, `contextHash` 일치 응답만 저장한다. 타임아웃, 검수 실패, `contextHash` 불일치 시 저장하지 않는다.
- 저장 컬럼은 `game_events.copy_protected`, `copy_revealed`와 모드별 해시 `copy_protected_context_hash`, `copy_revealed_context_hash`다. 모두 nullable이다. 모드별 컨텍스트가 달라 해시도 다르므로 단일 해시 컬럼을 공유하지 않는다.
- 저장 성공 시 scorer는 `game_updated`를 재발행한다. 클라이언트는 기존 재조회 흐름으로 정적 라벨이 AI 문구로 교체되는 경험을 제공한다.
- 이벤트 문구 소비는 `game_events` 직접 조회를 원천으로 한다. `AiCopyReader`는 `FINAL_HEADLINE` 전용이다.

### 이벤트 타임라인 노출 규칙

- 노출 판정은 서버 DTO 단계에서 `spoiler_level`로 강제한다. 보호 모드는 `PROTECTED_SAFE`만 노출하며, **미분류·알 수 없는 값은 모드와 무관하게 차단한다**(기본 차단).
- 보호 모드 응답: `{ eventId, eventType, inning, label, copy, observedAt }`. `label`은 §8의 보호 표기 문자열이고 `copy`는 AI 문구(nullable)다. 초/말·근거 수치·선수명·결과는 포함하지 않는다.
- 보호 모드 타임라인 본문은 `copy`를 우선 표시하고, `copy=null`이면 `label`을 폴백으로 표시한다. `label`은 타임라인 폴백 외의 용도로 쓰지 않는다(펄스 그래프 노드 미사용).
- 타임라인은 이닝별로 그룹화해 표시한다. 그룹 헤더는 보호 모드에서 이닝 숫자만, 공개 모드에서 초·말 구분까지 표시할 수 있다. 이벤트가 없는 이닝의 빈 그룹은 만들지 않는다.
- 공개 모드 응답: 보호 필드 + `inningType`, `players`, `evidence`, 공개 문구, `REVEALED_ONLY` 이벤트(득점·리드 변경·홈런·빅이닝).
- 산출·영속과 스키마는 예은, `GET /api/games/{id}/events` DTO·직렬화 가드는 민석 담당이다.
