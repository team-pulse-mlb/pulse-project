# 스포일러 노출 정책

## 1. 기본 원칙

| 원칙 | 정책 |
|---|---|
| 점수 비노출 | `watch_score`, `pregame_score`, `peak_base_score`는 숫자와 등급 모두 화면에 노출하지 않는다. |
| 서버 강제 | 보호 모드 필터링은 프론트가 아니라 API 응답 생성 단계에서 적용한다. |
| 결과 차단 | 점수, 승패, 리드 팀, 득점 결과, 결말을 직접 드러내지 않는다. |
| 짧은 설명 | 사용자는 계산 근거 전체가 아니라 추천 이유 태그와 종료 경기 한 줄 문구를 본다. |
| AI 제한 | AI는 추천 여부를 판단하지 않고, 종료 경기에서 서버가 전달한 안전한 context로 헤드라인·요약만 만든다. |

## 2. 모드별 노출 기준

### 보호 모드

보호 모드는 경기를 볼지 판단하는 데 필요한 맥락만 제공한다.

| 노출 가능 | 예시 |
|---|---|
| 매치업 | 팀 이름, 팀 약어 |
| 경기 상태 | 예정, 진행 중, 종료 |
| 진행 이닝 | 이닝 숫자 (초/말은 공격팀을 드러내므로 비노출) |
| 상황 맥락 | 아웃 카운트, 볼카운트, 득점권, 만루 |
| 추천 정보 | 추천 이유 태그, 종료 경기 AI 헤드라인·요약, 다시보기 구간 범위 |
| 표시 정보 | 경기장, 시작 시각, 관심 팀/선수 여부 |

팀 이름은 매치업 판단에 필요한 기본 정보이므로 보호 모드에서도 노출한다. 단, 어느 팀이 앞서거나 유리한지는 드러내지 않는다.

### 공개 모드

사용자가 경기 단위 공개 전환을 선택하면 아래 정보를 제공한다. 공개 전환은 확인 다이얼로그를 거치며, 해당 경기 상세에만 적용한다.

| 공개 가능 | 예시 |
|---|---|
| 점수 | 현재 점수, 최종 점수, play 전후 점수 |
| 결과 | 승패, 득점 play, 타석 결과, 원문 중계 텍스트 |
| 방향성 | 홈/원정 우세, 리드 변경 |
| 상세 매치업 | 타자/투수 이름, 이닝 초/말, play 타임라인 |

## 3. 경계 규칙

- **이닝 숫자는 노출, 초/말(`inningType`)은 비노출.** 이닝 숫자는 남은 관전 분량 판단에 필요하고 결과 방향을 드러내지 않는다. 초/말은 공격팀을 드러내 주자 압박과 결합 시 방향이 노출된다.
- **최근 흐름은 태그 변화 히스토리와 흥미 순간 이벤트 타임라인으로 제공한다.** play 전수를 마스킹해 나열하는 타임라인은 제공하지 않으며, 임계를 통과한 이벤트 마커만 노출한다.
- **다시보기 구간의 이닝 범위(예: 5회~7회)는 노출한다.** 초/말·play order·구간 점수는 숨긴다.
- **내부 추천 점수는 모드와 무관하게 항상 금지.** 공개 모드에서 공개되는 것은 "경기의 점수"이지 "추천 점수"가 아니다.

## 4. 화면별 노출 필드

| 화면 | 노출 | 숨김 |
|---|---|---|
| 홈 카드 (예정) | 매치업, 시작 시각, 구장, 선발 투수, 추천 태그 | (pregame_score는 정렬에만) |
| 상세 (예정) | 매치업, 시작 시각, 구장, 선발 투수, 추천 태그, 관심 선수 선발 여부 | 내부 점수·배당 일체 |
| 홈 카드 (진행) | 매치업, 이닝 숫자, LIVE 배지, 추천 태그 | 점수, 초/말, 아웃카운트 |
| 홈 카드 (종료) | 매치업, 다시보기 구간 수 배지, 추천 태그, AI 헤드라인 | 최종 점수, 승패 |
| 상세 (진행, 보호) | 매치업·선발, 이닝 숫자, 아웃·볼카운트, 득점권/만루 여부, 추천 태그, 태그 히스토리, 이벤트 타임라인(보호 안전 이벤트만), 관심 선수 출전 여부, `switchSuggestion` | 점수, 초/말, play 원문, 타석 결과, 공개 전용 이벤트 |
| 상세 (진행, 공개) | 보호 항목 + 점수, 이닝 초/말, play 타임라인(원문), 타석 결과, 이닝별 점수, 공개 전용 이벤트와 근거 수치 | 내부 추천 점수 |
| 상세 (종료, 보호) | 구간 목록(이닝 범위·구간 태그·AI 요약), 구간 수, 이벤트 타임라인(보호 안전 이벤트만) | 최종 점수, 승패, scoring_summary, 공개 전용 이벤트 |
| 상세 (종료, 공개) | + 최종 점수, 이닝별 점수, scoring_summary, 타임라인, 공개 전용 이벤트와 근거 수치 | 내부 추천 점수 |

홈 하단 전체 경기 목록 카드는 상태별로 위 홈 카드(예정·진행·종료)와 동일한 노출·숨김 기준을 적용한다. 라이브 수집 이전 종료 경기처럼 태그·구간이 없는 카드는 매치업만 표시한다.

## 5. 금지 필드 목록 (직렬화 가드 대상)

| 범위 | 금지 필드 |
|---|---|
| 전 모드 | `watch_score` · `base_score` · `pregame_score` · `peak_base_score` · `signal_contributions` 등 내부 점수 일체, odds 전체 |
| 보호 모드 | `runs`/`hits`/`errors`, 이닝별 점수, `inningType`(초/말), play `text`, `scoring_play`/`score_value`, 타석 `result`, `scoring_summary`, `home_score`/`away_score`, `game_events`의 `REVEALED_ONLY` 행 전체·`payload`·`inning_type` |

## 6. 금지 표현

보호 모드와 알림 문구에는 아래 표현을 사용하지 않는다.

| 금지 표현 | 이유 |
|---|---|
| 홈런 발생 | 구체적인 결과를 드러낸다. |
| 역전 성공 | 어느 팀의 상황인지 추측할 수 있다. |
| 끝내기 | 경기 결말을 드러낸다. |
| 실점 위기 | 특정 팀이 불리하다는 방향을 드러낸다. |
| 홈팀/원정팀 리드 | 앞서는 팀을 드러낸다. |
| 점수 차가 좁혀짐 | 점수 흐름과 방향을 드러낸다. |

## 7. 추천 이유 태그

태그는 내부 신호를 보호 모드에서 읽을 수 있는 짧은 표현으로 바꾼 값이다. AI 문구 생성 컨텍스트에도 내부 신호명이 아니라 이 표의 보호 표기 문자열만 전달한다.

| 내부 신호 | 보호 모드 태그 | 사용 위치 |
|---|---|---|
| `late_or_extra` | 후반 긴장 구간 | 카드, 상세, 다시보기 |
| `score_gap` | 접전 흐름 | 카드, 상세, 다시보기 |
| `recent_score` | 흐름 변화 | 카드, 상세, 다시보기 |
| `lead_change` | 흐름 급변 | 카드, 상세, 다시보기 |
| `big_inning` | 한 이닝 흐름 집중 | 카드, 상세, 다시보기 |
| `pressure` | 득점권 압박 | 카드, 상세, 다시보기 |
| `count_pressure` | 승부처 카운트 | 상세 |
| `early_slugfest` | 초반 난타 흐름 | 카드, 상세 |
| `pitcher_instability` | 투수 흔들림 | 상세 |
| `hard_contact` | 장타 위험 | 상세 |
| `favorite_team` | 관심 팀 | 카드, 알림 |
| `favorite_player` | 관심 선수 | 상세 |

상세 신호 중 `pitcher_instability`, `hard_contact`, `favorite_player`는 정렬 점수에 넣지 않고 태그와 상세 화면 표시에만 사용한다.

## 8. 흥미로운 순간 이벤트

경기 상세 타임라인에 표시하는 순간 단위 이벤트다. scorer가 라이브 계산 중 임계를 통과한 순간을 추출해 `game_events`에 영속하며, 종료 경기에도 그대로 재사용한다. 태그(§7)가 경기 단위 상태 표기라면 이벤트는 발생 시점이 고정된 점 단위 표기다.

| 이벤트 | 발생 조건(초기값) | 원천 | 등급 | 보호 표기 |
|---|---|---|---|---|
| `pressure_bases_loaded` | 만루 타석 | plays `runner_on_*` | 보호 안전 | 만루 승부 |
| `pressure_scoring_position` | 2·3루 주자 + 2아웃 | plays `runner_on_*` | 보호 안전 | 득점권 승부 |
| `long_at_bat` | 한 타석 8구 이상 | PA `pitch_number` | 보호 안전 | 긴 접전 승부 |
| `full_count_two_out` | 풀카운트 + 2아웃 | plays 카운트 | 보호 안전 | 승부처 카운트 |
| `pitcher_instability` | 선발 100구 이상 또는 구속 2mph 이상 하락 | PA `pitcher_pitch_count`·`release_speed` | 보호 안전 | 투수 흔들림 |
| `hard_contact` | `is_barrel` 또는 `exit_velocity >= 100` | PA | 보호 안전 | 강한 타구 |
| `scoring_play` | 득점 play | plays | 공개 전용 | — |
| `lead_change` | 앞선 팀 변경 | plays | 공개 전용 | — |
| `home_run` | 홈런 | plays | 공개 전용 | — |
| `big_inning` | 한 이닝 득점 play 2개 이상 | plays | 공개 전용 | — |

- 보호 표기에는 결과·방향성 표현(§6)을 결합하지 않는다. 선수명은 보호 모드에서 표기하지 않는다.
- 발생 임계와 경기당 상한(`hard_contact` 타석당 1회·경기당 8회 등)은 `scoring.yml`에서 관리한다. 상한은 타임라인 노이즈와 이벤트 밀집에 의한 간접 스포일러를 막기 위한 값이다.
- 관심 선수 타석은 사용자별 데이터이므로 전역 이벤트로 두지 않고 조회 시점에 계산한다.

### 이벤트 타임라인 노출 규칙

- 노출 판정은 서버 DTO 단계에서 `spoiler_level`로 강제한다. 보호 모드는 `PROTECTED_SAFE`만 노출하며, **미분류·알 수 없는 값은 모드와 무관하게 차단한다**(기본 차단).
- 보호 모드 응답: `{ eventType, inning, label, observedAt }`. `label`은 §8의 보호 표기 문자열이다. 초/말·근거 수치·선수명·결과는 포함하지 않는다.
- 공개 모드 응답: 보호 필드 + `inningType`, `payload` 근거 수치, 관련 선수, `REVEALED_ONLY` 이벤트(득점·리드 변경·홈런·빅이닝).
- 산출·영속과 스키마는 예은, `GET /api/games/{id}/events` DTO·직렬화 가드는 민석 담당이다.
