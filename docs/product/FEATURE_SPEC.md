# 기능 명세

사용자 흐름과 화면별 상세 동작(레이아웃, 상태, 예시 데이터)은 [USER_FLOW.md](USER_FLOW.md)를 따른다. 이 문서는 무엇을 만들지 범위를 정의하고, 정확한 임계값·필드 목록 같은 세부 설계는 `docs/design/`의 해당 문서를 단일 기준으로 따른다.

## 1. 핵심 용어

| 용어 | 정의 |
|---|---|
| 보호 모드 | 점수, 승패, 우세 팀, 결과 텍스트를 숨기는 기본 표시 모드 |
| 공개 모드 | 사용자가 전환을 선택해 해당 경기의 점수와 상세 결과를 보는 모드 |
| `watch_score` | 진행 중 경기의 내부 추천 점수 |
| `pregame_score` | 예정 경기의 내부 기대 점수 |
| `peak_base_score` | 종료 경기 정렬에 쓰는 라이브 중 최고 기본 점수 |
| `replay_segments` | 라이브 중 확정한 다시보기 추천 구간 |
| 추천 이유 태그 | 내부 신호를 보호 모드 문구로 바꾼 짧은 설명 |

## 2. 화면 구조

| 경로 | 화면 | 로그인 |
|---|---|---|
| `/login`, `/signup` | 로그인, 회원가입 | 불필요 |
| 최초 로그인 | 관심 팀 선택 온보딩 | 필요 |
| `/` | 홈 추천 보드 | 불필요 |
| `/games/:id` | 경기 상세, 공개 전환, 다시보기 | 불필요 |
| `/notifications` | 알림 센터 | 필요 |
| `/settings` | 관심 팀/선수, 알림 설정 | 필요 |

## 3. 기능 범위

### F1. 계정과 설정

- 이메일과 비밀번호로 가입·로그인한다.
- JWT 액세스 토큰과 리프레시 토큰을 사용한다.
- 관심 팀/선수, 알림 설정은 계정 단위로 저장한다.
- 비로그인 사용자는 홈 추천과 경기 상세를 볼 수 있다.
- 관심 팀 가산, 알림, 설정 관리는 로그인이 필요하다.

### F2. 홈 추천 보드

홈은 상단 추천 영역과 하단 전체 경기 목록으로 구성한다.

#### 상단 추천 영역

예정·진행 중·종료 경기를 상태별 추천 점수로 정렬해 최대 5장을 노출한다. 세 점수는 척도가 달라 하나의 랭킹으로 섞지 않고, 상태별 슬롯으로 채운다.

- 진행 중은 `watch_score`, 예정은 `pregame_score`, 종료는 `peak_base_score` 기준으로 정렬한다.
- 슬롯 채우기 순서는 진행 중 → 종료 → 예정이다. 진행 중 경기가 5장 이상이면 진행 중만으로 채운다. 남는 슬롯은 종료를 예정보다 우선 배치하되, 노출할 예정 경기가 있으면 최소 1장을 예정에 보장한다.
- 진행 중 경기의 랭킹 변경은 SSE로 반영한다. 로그인 사용자는 조회 시점에 관심 팀/선수 가산이 적용된 순서로 본다.
- 예정·진행·종료 경기는 홈 첫 화면에서 발견 가능한 시간 범위 안에서만 기본 노출한다(예정은 임박한 경기, 진행은 전체, 종료는 최근 종료 경기). 특정 경기 상세·다시보기는 `gameId` 기준으로 조회하며 이 시간 범위 필터와 별도로 동작한다.

#### 하단 전체 경기 목록

MLB 슬레이트(미국 동부시간 기준 날짜) 단위로 하루치 경기를 상태별로 보여준다.

- 날짜 필터: 기본값은 오늘 슬레이트다. 좌우 화살표로 전날·다음날 슬레이트로 이동하고, 달력 피커로 임의 날짜로 점프한다. 라이브 수집 이전 슬레이트도 조회할 수 있으며, 데이터가 없는 날짜는 빈 목록으로 표시한다.
- 상태 탭: 전체 · 예정 · 진행 · 종료.
- 정렬: 상태 탭(예정·진행·종료)은 추천순과 날짜순을 제공한다. 전체 탭은 상태가 섞여 추천 점수를 비교할 수 없으므로 시작 시각순으로만 정렬하며, 진행 중 경기를 상단에 고정한다.
- 추천순의 관심 팀/선수 가산은 상단 추천 영역과 동일하게 조회 시점에 적용한다.
- 라이브 수집 이전에 종료된 경기는 분석 결과가 없어 추천순 하단에 배치하고, 태그·구간 배지 없이 매치업만 표시한다.

스포일러 보호: 하단 목록 카드도 상단 카드와 동일한 보호 모드 기준으로 점수·승패·우세 팀·결과 텍스트를 숨긴다.

### F3. 스포일러 보호와 공개

- 모든 경기는 항상 보호 모드로 시작한다. 계정 단위 기본 공개 설정은 두지 않는다.
- 공개 전환은 경기 단위 전체 토글이다. 공개 상태는 서버에 저장하지 않고 클라이언트(localStorage)에만 둔다.
- 공개 전환은 별도 확인 절차 없이, 경기 상세 화면의 전환 버튼을 누르면 즉시 적용된다.
- 보호 모드에서는 점수, 승패, 우세 팀, 결과 텍스트, play 전후 점수, 이닝 초/말을 숨긴다.
- 매치업, 이닝 숫자, 주자 압박, 카운트, 추천 이유 태그는 보호 모드에서도 표시한다.
- 정확한 노출·숨김 필드 목록은 `SPOILER_POLICY.md`를 단일 기준으로 따른다.

### F4. 진행 중 경기 상세

보호 모드와 공개 모드 두 화면을 만든다. 보호 모드는 결과를 드러내지 않는 범위에서 관전 판단에 필요한 맥락(매치업, 이닝 숫자, 주자 압박, 추천 이유 태그, 태그 변화 히스토리, 흥미 순간 이벤트 타임라인, 관심 선수 출전 여부)을 보여주고, 공개 모드는 점수·이닝 초/말·play 타임라인·타석 결과 등 전체 정보를 보여준다. 정확한 노출·숨김 필드는 `SPOILER_POLICY.md`를 따른다.

- LIVE 경기는 SSE `game_updated` 신호를 받아 즉시 재조회로 갱신한다.
- `FINAL` 전이를 감지하면 다시보기 화면으로 전환한다.
- 다른 경기의 관전 가치가 더 높아지면 상세 응답의 `switchSuggestion`으로 전환을 안내한다.

### F5. 종료 경기 다시보기

- `replay_segments`를 이닝 범위 단위로 제공한다.
- 구간은 라이브 중 이미 확정한 결과를 사용한다.
- 보호 모드에서는 최종 점수와 승패를 숨기고 구간 태그와 AI 요약만 표시한다.
- 종료 경기 AI 헤드라인·구간 요약은 종료 경기에서만 노출한다. 예정·진행 중 경기 화면에는 AI 문구를 노출하지 않는다.
- AI 요약이 준비되지 않았을 때는 기본 문구를 표시한다.

### F6. 관심 팀과 관심 선수

- 관심 팀은 복수 선택할 수 있다.
- 관심 선수는 이름 검색으로 등록한다.
- 관심 팀은 정렬 가산, 태그, 알림에 사용한다.
- 관심 선수는 정렬 가산, 태그, 상세 화면 표시에 사용한다.

### F7. 알림

- 알림은 인앱 토스트와 알림 센터로 제공한다.
- 급상승 경기와 관심 팀 경기 시작만 알림 센터에 저장한다.
- 급상승 알림은 관심 팀과 무관하게 전역으로 발송하며, 설정에서 끌 수 있다. 관심 팀 경기 시작 알림은 해당 팀을 관심 팀으로 등록한 사용자에게만 간다.
- 급상승 알림은 추천 점수가 임계 이상으로 진입하고 최근 몇 분 사이 빠르게 상승했을 때 발동한다. 정확한 임계값·재무장 조건은 `NOTIFICATIONS.md`를 단일 기준으로 따른다.
- 경기 전환 안내는 상세 화면 토스트로만 제공한다.
- 알림 센터는 최신순 목록, 미읽음 배지, 읽음 처리, 7일 보관을 지원한다.

### F8. 설정

| 항목 | 값 | 기본값 |
|---|---|---|
| 관심 팀 | 복수 선택 | 없음 |
| 관심 선수 | 복수 선택 | 없음 |
| 관심 팀 경기 시작 알림 | 켬 / 끔 | 켬 |
| 급상승 경기 알림 | 켬 / 끔 | 켬 |
| 경기 전환 추천 | 켬 / 끔 | 켬 |

## 4. 범위 제외

| 항목 | 1차 대체 |
|---|---|
| Web Push | 인앱 토스트와 알림 센터 |
| 이닝 단위 부분 공개 | 경기 단위 전체 공개 |
| 소셜 로그인 | 이메일 기반 가입·로그인 |
| 관심 선수 개별 알림 | 상세 화면 표시와 정렬 가산 |
| 경기 내 이벤트별 알림 | 상세 화면 태그와 문구 |
| 관심 선수 부상 상태 표시 | 후속 (player_injuries 적재와 함께 도입) |
| 종료 경기 추천 표시 설정 | 종료 슬롯은 항상 다른 상태 슬롯과 동일한 규칙으로 채운다 |
| 선수 props, 구종별 매치업, 소셜 기능 | 후속 고도화 |

## 5. 통합 데모 완료 기준

1. 홈에서 라이브 경기 카드가 추천순으로 표시된다.
2. 카드 클릭으로 보호 모드 상세에 진입한다.
3. 상세 화면이 SSE 신호 수신 즉시 재조회로 갱신된다.
4. 공개 전환 후 점수와 타임라인이 표시된다.
5. 종료 경기에서 다시보기 구간과 AI 요약을 확인한다.
6. 가입 후 관심 팀 선택이 홈 정렬에 반영된다.
7. 급상승 경기 알림이 토스트와 알림 센터에 표시된다.
8. ai-service 장애 시 기본 문구로 화면이 정상 동작한다.
