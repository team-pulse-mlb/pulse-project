# 사용자 흐름과 화면 설계

## 1. 사용자와 진입 경로

| 사용자 | 로그인 | 진입 후 기본 경험 |
|---|---|---|
| 비로그인 방문자 | X | 홈에서 전체 추천을 보고, 경기 상세까지 자유롭게 본다. 관심 팀 가산·알림·알림 센터는 없다. |
| 신규 가입자 | 가입 완료 후 최초 로그인 | 회원가입에서 관심 팀과 알림 설정을 마치고, 최초 로그인 후 관심 선수 선택 온보딩을 거친 뒤 홈으로 이동한다. |
| 기존 로그인 사용자 | O | 홈 진입 시 관심 팀/선수 가산이 적용된 순서로 카드를 본다. |

## 3. 공통 UI 요소

### 3.1 공통 헤더

헤더는 모든 주요 화면 상단에 고정하고, 검은색 배경을 사용한다. 왼쪽에는 PULSE 로고를 두며, 클릭하면 홈(`/`)으로 이동한다. 오른쪽 영역은 로그인 상태에 따라 다르게 노출한다.

| 상태 | 오른쪽 영역 | 클릭 시 |
|---|---|---|
| 비로그인 | 로그인 | 로그인 모달 또는 로그인 화면 |
| 로그인 | 알림 아이콘, 마이페이지 아이콘 | 알림 센터(`/notifications`), 마이페이지(`/mypage`) |

알림 아이콘에는 읽지 않은 알림이 있을 때만 작은 배지를 표시한다. 배지는 개수 대신 미읽음 존재 여부만 드러내며, 점수·승패·팀 우세 같은 스포일러 정보를 포함하지 않는다.

마이페이지는 계정 정보 확인, 알림 설정, 관심 팀/선수 관리를 한 화면에서 처리한다(별도 설정 탭으로 나누지 않는다). 알림 설정은 마이페이지 안에서 인라인으로 켜고 끄며, 관심 팀/선수는 각 박스의 `추가`로 선택·관리 페이지에 진입한다(§4.10).

모바일에서는 로고를 왼쪽에 유지하고 오른쪽 아이콘은 알림, 마이페이지 순서로 최소 폭만 사용한다. 로그인 문구가 좁은 화면에서 넘치면 아이콘 또는 짧은 텍스트로 줄인다.

### 3.2 경기 상태 배지

경기 상태는 어느 화면에서든 같은 방식으로 표기한다.

| 상태 | 배지 텍스트 | 위치 |
|---|---|---|
| 예정 | `예정 · 7/9 08:05` 형태로 시작 시각 표기 | 카드 상단 |
| 진행 중 | `LIVE` + 이닝 숫자(예: `LIVE · 8회`) | 카드 상단, 상세 헤더 |
| 종료 | `종료` | 카드 상단 |

### 3.3 추천 카드 구조 (홈 상단/하단 공통)

카드는 상태 배지, 어웨이팀과 홈팀의 매치업, 상태별 핵심 정보 순서로 구성한다. 실제 배치는 홈 화면 캡처(§4.1)를 따른다.

| 상태 | 핵심 정보 | 예시 |
|---|---|---|
| 예정 | 구장, 선발 투수 | `Yankee Stadium · 선발 미확정` |
| 진행 중 | 최근 활성 태그 1개(`latestTag`, nullable) | `득점권 압박` |
| 종료 | AI 헤드라인. 없으면 `keyMoment` 1개. 둘 다 없으면 매치업만 표시 | `"7회 만루 승부가 등장한, 끝까지 긴장감이 이어진 경기입니다."` / `만루 승부` |

카드 전체가 클릭 영역이며, 클릭하면 항상 보호 모드로 경기 상세에 진입한다. 카드 자체에는 모드 전환 컨트롤을 두지 않는다(모드 전환은 상세 화면의 토글에서만 가능).

### 3.4 추천 이유 태그 칩

태그는 진행 중 카드에만 텍스트 칩 1개로 표기한다. 기준은 점수 기여도순이 아니라 가장 최근에 활성화된 태그(`latestTag`)다. 예정 카드는 태그를 표시하지 않고, 종료 카드는 태그 대신 AI 헤드라인 또는 `keyMoment`를 표시한다.

경기 상세 화면에는 태그 칩을 표시하지 않는다. 타임라인 리스트 본문은 AI 문구(`copy`)를 우선 표시하고 `copy`가 없으면 라벨을 폴백으로 표시한다. 라벨은 이 폴백 용도로만 쓰며, 펄스 그래프 노드에는 표시하지 않는다(§4.5). 정의된 태그 목록과 사용 위치는 SPOILER_POLICY.md §7을 따른다.

### 3.5 추천 사이드바 (경기 상세 전용)

경기 상세 화면 오른쪽에 홈 상단 추천 영역과 동일한 목록을 상시 패널로 보여준다. `switchSuggestion` 토스트(§3.6)와 역할이 다르다.

| 구분 | 추천 사이드바 | `switchSuggestion` 토스트 |
|---|---|---|
| 성격 | 상시 노출되는 탐색용 패널 | 실시간으로 튀어나오는 1회성 제안 |
| 내용 | 홈 상단 추천 목록 전체(최대 5장, 지금 보고 있는 경기는 제외) | 지금 경기보다 훨씬 더 볼 만해진 경기 1개 |
| 갱신 | 홈과 동일하게 `ranking_changed` 수신 시 조용히 재조회 | 상세 조회 응답의 `{ gameId, matchup, latestTag }` 값이 있을 때만 표시 |

데스크톱에서는 경기 상세 본문 오른쪽에 `다른 볼 만한 경기` 카드를 세로로 배치한다. 실제 배치는 경기 상세 캡처(§4.2~4.6)를 따른다.

- 데스크톱(오른쪽 컬럼 확보 가능)에서만 사이드바로 노출한다.
- 모바일에서는 오른쪽 컬럼이 없으므로 본문 하단에 "다른 볼 만한 경기" 섹션으로 내려서 붙인다(별도 화면 이동 없이 스크롤로 확인).
- 사이드바 카드도 §3.3 카드 구조·클릭 규칙을 그대로 따른다(클릭 시 보호 모드로 해당 경기 상세 진입).

### 3.6 토스트

| 종류 | 트리거 | 문구 예시 | 클릭 시 |
|---|---|---|---|
| 급상승 알림 | SURGE 이벤트 수신 | `지금 볼 만한 경기가 있어요 — 접전 흐름` | 해당 경기 상세로 이동 |
| 관심 팀 시작 알림 | 관심 팀 경기 진행 중 전환 | `관심 팀 경기가 시작됐어요 — SD @ CHC` | 해당 경기 상세로 이동 |
| 경기 전환 안내 | 상세 화면에서 `switchSuggestion` 수신 | `지금은 다른 경기가 더 볼 만해요 — 후반 긴장 구간` | 제안 경기 상세로 이동, 무시하면 자동 소멸 |

토스트 문구는 LLM을 사용하지 않는다. 서버가 태그별 고정 템플릿에 팀명과 `latestTag`를 치환해 완성 문자열 `message`로 내려주며, 프론트는 태그→문구 매핑을 갖지 않는다. 토스트는 화면 상단에 나타나 몇 초 후 자동으로 사라지며, 별도 확인 버튼을 요구하지 않는다.

### 3.7 모드 토글 (경기 상세 전용)

진행 중·종료 경기 상세 화면 상단에 `[보호 | 공개]` 세그먼트 토글을 상시 표시한다.

- 두 모드가 항상 함께 보이고, 현재 모드가 하이라이트된다(라벨이 바뀌는 단일 버튼이 아니다).
- 다른 쪽 세그먼트를 누르면 확인창(alert/confirm/dialog) 없이 그 자리에서 즉시 전환된다.
- 예정 경기 상세에는 토글을 노출하지 않는다(공개할 결과 자체가 아직 없음).
- 공개 상태는 이 브라우저(로컬)에만 경기 단위로 저장된다. 새로고침해도 유지되지만 다른 기기에는 적용되지 않으며, 카드에서 다시 진입하면 기본값은 보호 모드다(§F3).

## 4. 화면별 상세

### 4.1 홈 (`/`)

**목적**: 지금 볼 만한 경기를 가장 먼저 보여주고, 하루 전체 경기를 탐색할 수 있게 한다.

**레이아웃 — 상단 추천은 가로 스크롤 대신 비대칭(bento) 그리드로 배치한다.** 슬롯 채우기 순서(§F2, 진행 중 → 종료 → 예정)로 정해진 1번째 카드를 가로·세로 2배 크기의 히어로 카드로 키우고, 나머지 카드는 그 옆(데스크톱) 또는 아래(모바일)에 작은 카드로 채운다. 카드 크기 자체가 우선순위를 드러내며, 스크롤 없이 5장이 한 화면에 들어온다.

![홈 추천 그리드와 전체 경기 목록](./images/user-flow/home.png)

모바일 폭에서는 카드 1(히어로)이 전체 폭을 차지하고, 카드 2~5는 그 아래 2×2 그리드로 배치한다(가로 스크롤 없이 세로로만 이어진다).

**상태**

| 상태 | 표시 |
|---|---|
| 로딩 | 카드 자리에 스켈레톤 표시 |
| 상단 추천 없음 | 상단 영역 자체를 숨기고 하단 목록만 표시 |
| 하단 목록에 경기 없음 | 날짜별 탭은 해당 날짜의 빈 상태 안내. 예정 탭은 현재 이후 경기가 없을 때 "예정된 경기가 없어요" 안내 |
| SSE `ranking_changed` 수신 | 그리드 안에서 카드 위치를 부드럽게 재정렬(순간 이동이 아니라 위치 이동 애니메이션). 1순위가 바뀌면 새 1순위 카드가 히어로 크기로, 기존 히어로는 작은 카드 크기로 전환된다 |

**인터랙션**

- 카드 클릭 → 해당 경기 상세(보호 모드)로 이동.
- 날짜 화살표 클릭 → 하루 단위 이동, 달력 아이콘 클릭 → 임의 날짜 피커.
- 상태 탭 전환 → 목록만 갱신(상단 추천 영역은 유지).
- 정렬 드롭다운은 상태 탭이 "전체"일 때 비활성화(항상 시작 시각순 고정이라는 안내 텍스트 표시).

### 4.2 경기 상세 · 예정 (`/games/:id`, 예정)

![예정 경기 상세](./images/user-flow/game-scheduled.png)

예정 경기는 모드 토글(§3.7)을 노출하지 않는다. 데스크톱에서는 §3.5의 추천 사이드바가 본문 오른쪽에 붙는다.

### 4.3 경기 상세 · 진행 중 · 보호 모드 (기본 진입 상태)

**데스크톱 레이아웃**(본문 + 추천 사이드바, §3.5)

![진행 중 경기 보호 모드](./images/user-flow/game-live-protected.png)

현재 상황 영역은 상단 매치업 박스와 분리한 별도 섹션이다. 볼·스트라이크·아웃은 점등 표시(B/S/O)로, 주자 상황은 홈을 아래에 둔 다이아몬드로 시각화한다(주자 있는 베이스는 채운 마름모, 없는 베이스는 빈 마름모). 위 예시는 `situation`이 `runnerOnSecond=true`인 경우로 2루만 채운다. 매치업 박스에는 s/b/o·주자·현재 타석을 넣지 않는다. 진행 중 상세에는 펄스 그래프(긴장 곡선)를 표시하지 않는다(§4.5, 종료 경기 전용).

**모바일 레이아웃**(추천 사이드바는 본문 하단으로 이동)

본문 구성은 데스크톱과 같고, 추천 사이드바 카드만 본문 하단에 세로로 배치한다. 모드 토글은 상단에서 즉시 전환한다(§3.7).

노출 항목: 매치업, 이닝 숫자, 현재 상황 영역(볼·스트라이크·아웃 점등 + 주자 다이아몬드), 득점권/만루 여부, `경기 흐름`(보호 안전 이벤트 + AI 문구). 태그 칩, 태그 변화 히스토리, 별도 "흐름 변화" 섹션은 표시하지 않는다. 펄스 그래프(긴장 곡선)는 진행 중 상세에 표시하지 않고 종료 경기에서만 노출한다(§4.5). 점수·이닝 초/말·승패·play 원문·현재 타자/투수는 어떤 요소에도 없다(현재 타자의 소속팀이 곧 공격팀 정보이므로 보호 모드에서는 표기하지 않는다). 관심 선수 출전은 별도 박스로 두지 않으며, 현재 플레이 중인 선수를 표기하는 곳에서 강조로만 드러낸다(§4.4). 보호 모드는 현재 타자/투수를 표기하지 않으므로 관심 선수 강조도 보호 모드에서는 나타나지 않는다. 이닝 교대 중에는 현재 상황 영역에 `이닝 교대 중`을 대신 표기한다. 이 데스크톱/모바일 사이드바 배치는 §4.2·4.4·4.5·4.6 모든 상세 화면에 동일하게 적용되며, 이후 화면은 본문 영역만 표시한다.

### 4.4 경기 상세 · 진행 중 · 공개 모드

토글에서 `공개`를 누르면 다이얼로그 없이 그 자리에서 화면이 즉시 전환된다.

![진행 중 경기 공개 모드](./images/user-flow/game-live-revealed.png)

공개 모드는 보호 모드의 이벤트 목록을 그대로 확장하지 않는다. 같은 `경기 흐름` 영역의 데이터 원천을 최근 타석 결과로 교체하고, `plays.text`를 LLM이 한국어로 번역한 문구를 최신순으로 표시한다. 번역 문구는 프론트에서 재요약·재조합하지 않는다. 점수, 이닝 초/말, 현재 타자/투수, 타석 결과, 이닝별 점수도 함께 보여주되 보호 이벤트와 공개 전용 이벤트 문구는 중복 표시하지 않는다. 현재 타석(타자 vs 투수)은 상단 매치업 박스가 아니라 현재 상황 영역에 s/b/o·주자 다이아몬드와 함께 둔다. 현재 타석의 타자·투수나 최근 플레이에 관심 선수(`favoritePlayersPlaying`)가 등장하면 별도 박스 대신 그 자리에서 강조 표기한다. 이닝 교대 중에는 현재 타석 줄을 표시하지 않는다(주자 다이아몬드는 유지). 진행 공개 모드에도 펄스 그래프는 표시하지 않는다. 전환은 §3.7 토글 규칙을 따른다.

### 4.5 경기 상세 · 종료 · 보호 모드

![종료 경기 보호 모드](./images/user-flow/game-final-protected.png)

**긴장 곡선 규칙** (전체 정책은 SPOILER_POLICY.md §5 예외 조항)

- 종료 경기에서만 표시한다. 진행 중 상세(보호·공개)와 예정 화면에는 곡선 영역을 두지 않는다.
- 종료 보호 모드는 이닝 단위, 종료 공개 모드는 하프이닝 단위 해상도로 그린다. 서버가 1~5단계로 양자화한 레벨 값만 내려주며, 세로축에는 눈금·숫자를 표기하지 않는다(가로축 이닝 라벨은 허용).
- 곡선에는 이벤트 노드를 얹지 않는다. 곡선은 이닝별 긴장 수준만 표현하고, 보호 모드의 사건 표시는 아래 `경기 흐름` 목록이 전담한다. 곡선 포인트 탭으로 목록을 이동시키는 연결도 두지 않는다.
- AI 헤드라인은 모드마다 별도로 생성한 문구다. 보호 모드 헤드라인은 결과를 드러내지 않는 안전 문구이고, 공개 모드 헤드라인은 최종 점수·승패를 반영한 문구다(§4.6 예시 참고). **기본 문구는 없다** — 헤드라인이 아직 없으면 헤드라인 영역 자체를 렌더링하지 않으며, 나머지 영역의 높이·배치는 그대로 유지한다. 생성이 완료되면 `game_updated` 재수신 시 빈 자리에 문구가 나타난다.
- 보호 모드의 `경기 흐름` 본문은 이벤트 AI 문구(`copy`)를 우선 표시하고, 문구가 없으면 라벨을 폴백으로 표시한다. 목록은 이닝별 그룹 헤더로 묶어 표시하며, 이벤트가 없는 이닝의 빈 그룹은 만들지 않는다. 공개 모드의 같은 영역은 이벤트 대신 번역된 최근 플레이를 표시한다.

**상태**

| 상태 | 표시 |
|---|---|
| AI 헤드라인 없음 | 헤드라인 영역 미렌더링(레이아웃 유지) |
| 곡선 데이터 없음(점수 이력 미확보 경기) | 곡선 영역 미렌더링, 현재 모드의 `경기 흐름`만 표시 |

### 4.6 경기 상세 · 종료 · 공개 모드

토글에서 `공개`를 누르면 즉시 전환된다.

`경기 흐름`에는 보호 이벤트 타임라인을 표시하지 않고, 공개 모드용 최근 플레이 번역 목록을 표시한다. 득점 play 목록은 전체 최근 흐름이 아니라 득점 기록만 모은 별도 영역으로 유지한다.

![종료 경기 공개 모드](./images/user-flow/game-final-revealed.png)

### 4.7 회원가입 Step 2 · 관심 팀 선택

![회원가입 관심 팀 선택](./images/user-flow/onboarding.png)

관심 팀은 최대 3팀까지 선택할 수 있다. 선택 없이 건너뛸 수 있으며, 다음 단계인 알림 설정으로 이동한다. 가입 후에는 설정에서 언제든 변경할 수 있다.

### 4.8 로그인 / 회원가입 (`/login`, `/signup`)

![로그인](./images/user-flow/login.png)

회원가입은 계정 정보 입력, 관심 팀 선택, 알림 설정의 3단계로 진행한다. 마지막 단계에서 가입을 완료한 뒤 로그인 화면으로 이동한다. 로그인은 이메일·비밀번호 입력 폼과 회원가입 전환 링크를 제공하며, 소셜 로그인은 제공하지 않는다.

### 4.9 최초 로그인 온보딩 · 관심 선수 선택 (`/onboarding`, 로그인 필요)

신규 회원은 가입 완료 후 로그인한다. 최초 로그인인 경우에만 관심 선수 선택 온보딩으로 이동하고, 관심 선수 선택을 마치면 홈으로 이동한다. 이후 로그인부터는 온보딩을 반복하지 않고 바로 홈으로 이동하며, 관심 선수는 설정에서 언제든 변경할 수 있다. 선수 검색 API는 로그인 사용자만 호출할 수 있다.

### 4.10 마이페이지 (`/mypage`, 로그인 필요)

![마이페이지](./images/user-flow/mypage.png)

마이페이지는 별도 탭 이동 없이 한 화면에서 계정 정보, 알림 설정, 관심 팀/선수를 함께 보여준다.

- 상단은 알림 설정 박스로, 관심 팀 경기 시작·급상승·경기 전환 추천 토글을 인라인으로 바로 켜고 끈다(설정 화면으로 이동하지 않는다).
- 그 아래 관심 팀 박스와 관심 선수 박스에 현재 등록 목록을 칩으로 보여준다. 각 박스의 `+ 추가`를 누르면 팀/선수 선택·관리 페이지로 진입해 등록·삭제한다.
- 계정 이메일과 로그아웃은 상단에 둔다. 로그아웃 후에는 홈으로 이동한다.

### 4.11 알림 센터 (`/notifications`, 로그인 필요)

![알림 센터](./images/user-flow/notifications.png)

`●`는 미읽음, `○`는 읽음. 항목 클릭 시 읽음 처리와 함께 해당 경기 상세로 이동한다. 최신순으로 정렬하고 7일이 지난 항목은 노출하지 않는다.

### 4.12 관심 팀/선수 관리 (`/settings`, 로그인 필요)

마이페이지 관심 팀/선수 박스의 `추가`로 진입하는 선택·관리 페이지다. 알림 토글은 마이페이지(§4.10)로 옮겼으므로 이 화면에는 두지 않는다.

![관심 팀과 선수 관리](./images/user-flow/settings.png)

관심 팀은 회원가입 Step 2와 동일한 팀 선택 UI(최대 3팀)를 사용하고, 관심 선수는 이름 검색으로 등록한다. 등록·삭제 후 마이페이지로 돌아가면 갱신된 목록이 반영된다.

종료 경기 추천 표시 여부 설정은 없다 — 종료 슬롯은 다른 상태와 동일한 규칙으로 항상 채워진다. 스포일러 공개 모드의 계정 단위 기본값 설정도 없다 — 공개 상태는 브라우저 로컬에만 저장된다(§3.7, §F3).

## 5. 인터랙션 규칙 모음

| 규칙 | 내용 |
|---|---|
| 모드 토글 | 진행 중·종료 상세 상단에 `[보호 | 공개]` 세그먼트 토글이 상시 표시된다(예정 경기 제외). 두 모드가 항상 함께 보이고 현재 모드가 하이라이트되며, 다른 쪽을 누르면 확인창 없이 즉시 전환된다. |
| 카드 → 상세 진입 | 항상 보호 모드로 진입한다. 직전에 같은 경기를 공개 모드로 봤어도 카드에서 다시 들어오면 보호 모드부터 시작한다(§F3, 경기 단위 로컬 저장이지만 진입 시 기본값은 보호). |
| 실시간 갱신 | 홈은 `ranking_changed`, 상세는 `game_updated` 수신 시 화면이 알림 없이 조용히 재조회되어 갱신된다(로딩 스피너 전체 화면 노출 없음). |
| FINAL 전이 | 상세 화면을 보던 중 경기가 종료되면 자동으로 다시보기 화면 구성(§4.5)으로 바뀐다(페이지 이동이 아니라 같은 화면 내 콘텐츠 전환). 종료 직후 AI 헤드라인은 비어 있을 수 있으며, 생성·저장이 완료되면 `game_updated` 재발행으로 빈 자리에 문구가 나타난다. |
| 경기 전환 안내 | 토스트로만 제공하며 자동 이동은 하지 않는다. 사용자가 클릭해야 이동한다. |
