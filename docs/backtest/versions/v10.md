# scoring v10 — 리드 변경 신호를 시간 지수 감쇠로 전환

- 날짜: 2026-07-18
- PR: #262 (`feat/yeeun-lead-change-decay`, 커밋 `3b904fc`)
- 기준 버전: v9

## 변경 상수

| 키 | v9 | v10 |
|---|---:|---:|
| `lead-change.tau-seconds` | (없음) | 300 |

코드 변경을 동반한 버전이다. 리드 변경 보너스가 고정 윈도(`window-plays: 16`) 동안 일정하게 유지되던 방식에서, 발생 시점부터 시간 지수 감쇠(τ=300초)하는 방식으로 바뀌었다. `window-plays`는 벽시계 시각이 없는 `S3_BACKFILL` 재생의 근사 윈도로 남는다.

## 변경 가설

- 고정 윈도 방식은 리드 변경 후 16 plays 동안 점수가 계단식으로 유지되다 한 번에 사라져, 긴장이 식는 흐름을 반영하지 못했다.
- `recent-score`(τ=180초)와 같은 시간 감쇠 계열로 통일하면 라이브 펄스 그래프의 곡선이 자연스러워지고, 관측 주기 차이에 따른 점수 왜곡도 줄어든다.

## 검증 결과

- [impact_v9_vs_v10_2026-07-04_2026-07-17](../impact_v9_vs_v10_2026-07-04_2026-07-17.md): 3경기(OPERATIONAL 1,222 plays), AUC 0.5353→0.5353, peak 순위 상관 1.0, 알림 0건.
- **표본 3경기로 대표성이 없다.** 대상 기간의 운영 수집분이 적었던 시점의 실행이라, 기간을 넓힌 재실행이 필요하다.

## 재검증 (2026-07-19, 긴장 이벤트 라벨 재정의 후)

라벨 재정의(PR #273) 병합 후 기간을 넓혀 자기 검증을 재실행하고, 재정의된 지표의 기준선을 확보했다.

- [impact_v10_vs_v10_2026-07-05_2026-07-18](../impact_v10_vs_v10_2026-07-05_2026-07-18.md): 101경기, OPERATIONAL 44,672 plays.
- **AUC(상태 신호) 0.6928, AUC(전체 신호) 0.6761** — 이후 튜닝의 비교 기준값. 정렬 품질 Spearman 0.3839, 일평균 알림 4.78건.
- 라벨 정의가 바뀌었으므로 이전 AUC(v6 자기 검증 0.5597, 위 3경기 실행 0.5353)와 직접 비교하지 않는다.
- τ 민감도: 같은 기간·같은 조건에서 τ=150, 600 후보를 비교한 결과 AUC(전체 신호) 델타 ±0.0001, 나머지 지표 변화 없음. 리드 변경은 사후 반응 신호로 분류돼 상태 AUC에서 제외되고 전체 AUC에도 사실상 중립이므로, τ는 현행 300을 유지한다.

## 판단

- 채택(코드 전환 자체가 목적, 상수는 초기값). τ=300은 리드 변경 여운을 recent-score(180초)보다 길게 두는 초기 추정치다.
- 후속(완료): 기간을 넓힌 백테스트로 τ 민감도 확인, AUC 지표 재정의(긴장 이벤트 라벨) 후 재평가 → 위 재검증 절에 기록.
